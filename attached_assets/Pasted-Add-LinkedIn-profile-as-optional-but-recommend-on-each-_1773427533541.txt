Great question.
If you want the platform to feel credible and high-signal, every profile type should only become “Active / Discoverable” after reaching a minimum completeness level.

Think of it like LinkedIn profile strength, but designed for a startup ecosystem.

A good rule: 70–80% completion required to activate profile.

Below is a complete activation checklist for each profile type.

⸻

🌍 GLOBAL PROFILE ACTIVATION SYSTEM

Each profile has:
	•	Draft → profile being built
	•	Incomplete → below activation threshold
	•	Active → visible in discovery
	•	Verified (future feature)

⸻

🟢 STARTUP PROFILE ACTIVATION REQUIREMENTS

Startup profiles are the core of the platform, so the requirements should be stricter.

Basic Identity

Required fields:
	•	Startup Name
	•	Logo
	•	One-line Pitch
	•	Industry / Sector
	•	Startup Stage
	•	City
	•	Country
	•	Founded Year
	•	Website

⸻

Startup Overview

Required:
	•	Problem statement
	•	Solution description
	•	Market opportunity (TAM / SAM / SOM)

Optional but recommended:
	•	Why Now

⸻

Traction / Metrics

One of the following must be filled:

Option A (Revenue stage)
	•	MRR or ARR
	•	Growth rate

Option B (Pre-revenue)
	•	Total users
	•	Key milestone

⸻

Team

Minimum:
	•	At least 1 founder profile

Each founder must include:
	•	Name
	•	Role
	•	LinkedIn profile
	•	Short bio

⸻

Funding Round

Required to activate:
	•	Round Type
	•	Target Raise
	•	Funding Instrument (SAFE / Equity etc)

Optional but recommended:
	•	Valuation
	•	Minimum ticket size
	•	Close date

⸻

Documents

At least 1 pitch asset required

Options:
	•	Pitch Deck
OR
	•	Pitch Video

Optional:
	•	Financial Model
	•	Data Room
	•	Cap Table

⸻

Final Activation Rule

Startup profile becomes Active when:

✔ 80% profile completion
✔ At least 1 founder added
✔ Funding round created
✔ Pitch deck uploaded

Alright, let’s design a clean, scalable database schema that supports all four user types on your platform:
	•	Startups (Founders)
	•	Investors
	•	Service Providers
	•	Startup Enthusiasts

This schema is structured for PostgreSQL + Prisma / SQL style, and it’s designed so the platform can scale globally without needing a major rebuild later.

The philosophy:
👉 One unified user system + role-specific profile tables.

⸻

🧩 CORE ARCHITECTURE

Users (base identity)
│
├── Founder Profiles
│   └── Startups
│       ├── Startup Overview
│       ├── Metrics
│       ├── Funding Rounds
│       ├── Documents
│       └── Team
│
├── Investor Profiles
│
├── Service Provider Profiles
│
└── Enthusiast Profiles

This allows a single login system, but different profile types.
